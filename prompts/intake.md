# 导师基础信息录入脚本（含预蒸馏模板开关）

## 开场白

```
我来帮你创建这位导师的 Skill。先选工作模式和蒸馏策略，再回答基础问题（导师代号必填）。
```

---

## 问题序列

### Q0：务实模式开关（必选）

```
你希望这个导师默认采用哪种策略？

[A] 学术理想型
    优先方法严谨与长期学术价值，保守推进

[B] 毕业优先型（含合理裁缝）
    在导师红线内最大化实用性，优先按期毕业与可交付
```

记录字段：
- `working_mode`: `academic_ideal` 或 `graduation_first`

解释原则：
- “合理裁缝”仅指 **问题重述、实验组织、叙事结构优化、优先级取舍**
- 不包含伪造、篡改、抄袭、虚构引用、隐瞒关键负结果等违规行为

---

### Q0.5：蒸馏策略开关（必选）

```
这次蒸馏，你要用哪种策略？

[A] strict_distill（纯素材蒸馏）
    只用你提供的素材，不补通用模板

[B] hybrid_distill（素材 + 预蒸馏方法论，推荐）
    在保留导师个性的前提下，补齐可执行方法

[C] template_first（模板优先）
    素材很少时先用预蒸馏框架，后续再逐步替换
```

记录字段：
- `distill_strategy`: `strict_distill` / `hybrid_distill` / `template_first`
- `use_template_methodology`: `false`（A）或 `true`（B/C）

---

### Q1：导师代号（必填）

```
这位导师怎么称呼？建议用代号（保护隐私），例如：李导、A教授、老板A。
```

规则：
- 接受中文、英文或混合输入
- 生成 slug 时统一转小写并使用 `-` 连接
- 中文尽量转拼音（如可用），否则保留 ASCII 字符并去空格

---

### Q2：基本信息（一句话）

```
用一句话描述 ta 的基本信息：院校、学科、职称、团队规模、研究方向（想到什么写什么）。

例：某985 计算机学院 教授 博导 带12个研究生 做多模态与具身智能
```

解析字段（缺失留空）：
- 院校（school）
- 学科（discipline）
- 职称（title）
- 团队规模（team_size）
- 研究方向（research_direction）

---

### Q3：指导风格（一句话）

```
用一句话描述 ta 的指导风格：管理标签 + 口头禅 + 你的主观印象。

例：高标准低容错，喜欢说“先把问题定义清楚”，改稿非常细，周末会催进度。
```

提取内容：
- 指导风格标签（advising_style）
- 常用话术 / 口头禅（catchphrases）
- 情绪触发点（triggers）
- 主观印象（impression）

---

### Q4：素材充分度自评（可选，默认系统估算）

```
你觉得素材充分度大概多少分（0-100）？
- 0-30：非常少，几乎没原话
- 31-60：有一些素材，但覆盖不完整
- 61-100：素材较多，能反映真实风格
```

记录字段：
- `material_sufficiency_score`（0-100）

系统估算建议（没有用户打分时）：
- 依据素材字数、覆盖场景数量、时间跨度给一个 0-100 分
- 估算规则可简化为：
  - 字数 < 2k 且场景 < 3：<40
  - 字数 2k-10k 或场景 3-6：40-70
  - 字数 > 10k 且场景 > 6：>70

## 汇总确认模板

```
信息汇总：

  👤  代号：{name}
  🏫  {school} {discipline} {title}
  👥  团队规模：{team_size}
  🔬  研究方向：{research_direction}
  🏷️  指导风格：{advising_style}
  💬  口头禅：{catchphrases}
  ⚙️  模式：{working_mode}
  🧩  策略：{distill_strategy}
  🧠  预蒸馏模板：{use_template_methodology}
  📊  素材分：{material_sufficiency_score}
  🧠  印象：{impression}

确认无误？（确认 / 修改 [字段名]）
```

确认后进入素材导入阶段。
