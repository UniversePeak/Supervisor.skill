# Academic Style 分析指南

## 目标

从原材料中提取“导师在学术指导上的稳定模式”，生成可执行规则，而不是情绪化标签。

---

## 分析输入

- 用户填写的基础信息（院校/学科/职称/方向）
- 用户选择的工作模式（academic_ideal / graduation_first）
- 导师相关材料（聊天、邮件、批注、组会纪要、培养方案、课题文档）
- 用户补充口述

---

## 核心维度

### 1. 选题与问题定义
- 导师偏好的选题来源（应用驱动 / 理论驱动 / 数据驱动）
- 题目筛选标准（创新性、可行性、周期、资源匹配）
- 常见否决理由（“问题不真”“工作量不可控”“缺 baseline”）

### 2. 方法论偏好
- 常用研究路径（先复现再创新 / 先搭系统后写论文）
- 对 SOTA 的态度（追热点 / 重扎实）
- 对实验深度和对照实验的要求

### 3. 论文标准
- 摘要与引言要求
- 图表、实验、消融的最低标准
- 对 Related Work 和讨论部分的关注点
- 最常见改稿意见（按频次排序）

### 4. 过程管理
- 里程碑粒度（周计划 / 双周计划 / 月计划）
- 催办风格（软催 / 高频跟进 / 截止点强压）
- 组会组织方式（逐个过稿 / 主题汇报 / 快速轮询）

### 5. 资源与协作
- 对师兄师姐协作的期待
- 对跨组合作、共一排序、数据共享的态度
- 对工具链和基础设施的偏好

### 6. 风险红线
- 学术伦理底线（禁止行为）
- 对延期、摆烂、重复劳动的容忍度
- “必须及时上报”的风险类型

### 7. 模式差异化策略
- 学术理想型：长期价值、方法完整性、可解释性优先
- 毕业优先型：在红线内优先完成毕业要求、最小必要工作量、交付节奏可控
- 同一问题下给出两套路径时，明确资源成本与风险成本

---

## 输出格式

请输出以下结构：

```markdown
## Academic 分析结果

### A. 高置信结论（有明确证据）
- ...

### B. 中置信结论（证据有限但一致）
- ...

### C. 待确认假设（需用户确认）
- ...

### D. 关键原话证据（最多 8 条）
- [来源+时间] 原话/批注

### E. 模式映射建议
- academic_ideal 下建议：
- graduation_first 下建议：
```

---

## 约束

1. 不得编造导师原话或实验结果
2. 结论优先“可执行”而非“形容词”
3. 若信息不足，明确标注 `[待补充]`
4. 学术伦理相关条目必须单列
