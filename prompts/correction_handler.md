# 纠偏处理指南（correction_handler）

## 适用触发

当用户表达以下意图时使用：
- "不对"
- "他不会这样说"
- "他应该是..."
- "这句太温柔了/太狠了"

---

## 处理流程

### 1. 抽取纠偏三元组

从用户输入中提取：
- `wrong`: 当前输出哪里不对
- `correct`: 应该改成什么
- `scene`: 场景（如组会、改稿、催进度、开题）

### 2. 判定归属

- 如果涉及学术规则、流程、标准 → 归到 `academic.md`
- 如果涉及语气、风格、情绪、关系行为 → 归到 `persona.md`
- 如果涉及毕业策略模板（如数据表达优化、审稿最小改动）→ 归到 `playbook.md`
- 无法判定时，默认归到 `persona.md` 并提示用户确认

### 3. 生成 correction 记录

统一格式：

```markdown
- [{scene}] 不应该：{wrong}；应该：{correct}（来源：用户纠偏，{date}）
```

### 4. 写入策略

- 追加到对应文件的 `## Correction 记录`
- 若不存在该标题，则先创建章节再写入
- 同步更新 `meta.json.corrections_count`

---

## 约束

1. 纠偏优先于旧结论
2. 不删除历史记录，保留可追溯性
3. 纠偏后重新生成 `SKILL.md`，确保立即生效
