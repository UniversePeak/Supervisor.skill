# 增量合并指南（merger）

## 目标

在不覆盖既有结论的前提下，把新素材增量并入 `method_core.md`、`academic.md`、`persona.md` 与 `playbook.md`。

---

## 步骤

### Step 1：读取当前版本

- 读取现有 `academic.md`
- 读取现有 `persona.md`
- 读取现有 `playbook.md`
- 读取现有 `method_core.md`
- 读取用户新素材

### Step 2：做差异归类

把增量信息分为：

1. **新增事实**：之前没有的信息，直接追加
2. **强化证据**：支持已有结论，补证据或提高置信度
3. **冲突信息**：与已有结论冲突，记录冲突并提示确认
4. **无关噪声**：与导师画像无关，不并入

### Step 3：分区合并

- Academic 相关增量合并到对应章节：
  - 选题规则 / 执行规范 / 论文标准 / 组会机制 / 红线
- Persona 相关增量合并到对应 Layer：
  - Layer 1~4 或 Correction 记录
- Playbook 相关增量合并到对应模板：
  - 数据表达优化 / 故事化写作 / 最小改动审稿 / 保底路线
- Method Core 相关增量合并到方法论章节：
  - 任务拆解 / 决策优先级 / 风险分级 / 输出格式

### Step 4：版本更新

- 先执行版本备份
- 再写入增量
- 更新 `meta.json` 的 `version`、`updated_at`、`sources`
- 若 `distill_strategy` 为 `strict_distill`，仅在用户明确允许后写入模板来源内容

---

## 冲突处理规则

当出现冲突时，使用以下优先级：

1. 时间更近的原话证据
2. 多源一致证据（聊天 + 批注 + 组会）
3. 用户显式纠正

若仍无法决策，保留两种说法并标注 `[待用户确认]`。

---

```markdown

- 新增：{n} 条

- 冲突：{n} 条

- ...

## Persona 增量
- ...

## Playbook 增量
- ...

- ...

## 待确认冲突
- ...
```
