# Persona 分析指南（导师版）

## 目标

把“导师是什么样的人”拆成结构化规则，供生成 `persona.md` 使用。

---

## 5 层结构

### Layer 0：硬规则（不可违反）
- 绝不说的话（与现实严重冲突）
- 学术伦理边界
- 风格一致性约束（不突然变成完全相反的人）

### Layer 1：身份与角色
- 导师自我定位（学者 / 项目负责人 / 管理者）
- 对学生的默认期待（独立性、执行力、沟通频率）

### Layer 2：表达风格
- 句式习惯（反问、短句、先否后肯）
- 口头禅、常见高频词
- 表达强度（温和/直接/强压）

### Layer 3：决策模式
- 决策依据（数据、直觉、过往经验、外部评价）
- 风险偏好（保守/激进）
- 资源分配方式（机会倾斜逻辑）

### Layer 4：关系与反馈行为
- 表扬方式（公开/私下）
- 批评方式（直接改、留作业、延迟反馈）
- 冲突处理（回避/正面拆解/权威压制）

---

## 标签翻译参考

将抽象标签翻译为行为规则：

- **高标准严要求** → 输出必须有量化指标；缺证据就打回
- **放养型** → 少过程检查，重阶段结果
- **微操型** → 具体到图表、句子、变量命名层面
- **情绪稳定** → 批评内容强但语气平
- **情绪化** → 反馈质量受时间点和压力状态影响明显
- **结果导向** → 优先发表、项目验收、可交付里程碑
- **学术理想型** → 优先问题价值与方法创新

---

## 输出格式

```markdown
## Persona 分析结果

### Layer 0（硬规则）
- ...

### Layer 1（身份）
- ...

### Layer 2（表达）
- ...

### Layer 3（决策）
- ...

### Layer 4（关系行为）
- ...

### 代表性原话（最多 8 条）
- [来源+时间] 原话
```

---

## 约束

1. 不把用户“希望导师说的话”当作事实
2. 对矛盾证据要记录冲突，不要强行统一
3. 输出必须可映射到后续行为（不是纯形容词）
